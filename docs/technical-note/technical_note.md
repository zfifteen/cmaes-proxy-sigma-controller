# Technical Note: Mechanism-Focused Sigma Control in `cmaes-proxy-sigma-controller`

## Abstract
This technical note documents a mechanism-focused external sigma controller for CMA-ES. The controller is designed to manage constrained-regime occupancy while preserving optimizer-internal independence. Claims in this note are intentionally bounded to repository-contained evidence: controller contract specification, deterministic policy/adapter tests, and reproducible reference-run artifacts generated by the publication evidence workflow.

## 1. Problem Framing and Claim Boundaries

### 1.1 Problem
In noisy optimization, sigma control can fail in two ways:
- persistent over-expansion (wasted exploration),
- early or persistent clamp-dominant contraction (lost adaptive window).

This project isolates sigma governance as an external control layer so that CMA-ES internals remain untouched.

### 1.2 Scope
In scope:
- deterministic external sigma policy,
- clamp-safe control updates,
- occupancy-aware telemetry (`fraction_at_floor`, entries/exits, phase transitions),
- post-`tell` adapter sequencing.

Out of scope:
- modifying covariance/recombination/evolution-path internals,
- restart orchestration,
- universal superiority claims across all objective families.

## 2. Mechanism Model

### 2.1 Signal, noise, and smoothing
For minimization per generation:
- `signal_t = max(best_so_far_(t-1) - current_best_t, 0)`
- `noise_t = 1.4826 * MAD(fitness_t) + noise_floor_t`
- `snr_t = signal_t / noise_t`
- `ema_t = alpha * snr_t + (1 - alpha) * ema_(t-1)` (or first-observation initialization)

Normative definitions are specified in [docs/TECHNICAL_DESIGN_SPEC.md](/Users/velocityworks/IdeaProjects/cmaes-proxy-sigma-controller/docs/TECHNICAL_DESIGN_SPEC.md) Section 7.1.

### 2.2 Clamp regime and action selection
Base multiplicative action:
- down if `ema_t < snr_down_threshold`,
- up if `ema_t > snr_up_threshold`,
- neutral otherwise.

Sigma is then clipped to:
- `floor_sigma = initial_sigma * sigma_min_ratio`,
- `ceiling_sigma = initial_sigma * sigma_max_ratio`.

The controller records whether clipping changed the action proposal (`was_clamped`), enabling explicit constrained-regime telemetry.

### 2.3 Phase transitions and bounded recovery
Controller phases are:
- `WARMUP`,
- `ACTIVE`,
- `CONSTRAINED`,
- `RECOVERY`.

Transition ordering and precedence are normative in the technical spec (Section 7.4). Recovery is constrained by minimum floor streak and cooldown, preventing uncontrolled oscillation while allowing bounded de-constraining behavior.

## 4. Threats to Validity and Explicit Non-Claims

### 4.1 Threats to validity
- Evidence here emphasizes contract correctness and deterministic behavior, not broad empirical superiority across benchmark suites.
- Reference runs are intentionally lightweight and fixed-seed; they are reproducibility artifacts, not comprehensive performance studies.
- Cross-runtime floating-point or dependency drift can alter numeric outcomes outside the strict replay tier.

### 4.2 Explicit non-claims
- No claim that this controller universally outperforms vanilla CMA-ES.
- No claim that constrained-regime occupancy metrics alone identify globally optimal sigma policy.
- No claim that this external policy subsumes optimizer-internal adaptation mechanisms.
